# Contributing to Awesome Free Developer Resources

Thank you for taking the time to contribute! 🎉

This list exists to help developers worldwide find the best free tools and resources. Every contribution matters.

---

## Before You Contribute

- Check if the resource is **already listed**
- Make sure it has a **genuine free tier** (not just a free trial)
- Make sure it is **useful to developers globally** (not region-locked)

---

## How to Add a Resource

1. **Fork** this repository
2. **Edit** `README.md`
3. Add your resource in the correct category using this format:

```
- [Tool Name](https://example.com) — One line description of what it does.
```

4. **Open a Pull Request** with the title:
```
Add [Tool Name] to [Category Name]
```

---

## Format Rules

- ✅ Use this exact format: `- [Name](link) — Description.`
- ✅ Description must be **one line, under 15 words**
- ✅ Link must go to the **official website or repo**
- ✅ Alphabetical order within each category (when possible)
- ❌ No referral or affiliate links
- ❌ No paid-only tools
- ❌ No duplicate entries
- ❌ No self-promotion without clear disclosure in PR description

---

## What Qualifies as "Free"

| ✅ Allowed | ❌ Not Allowed |
|---|---|
| Free tier with no expiry | 14-day free trial only |
| Open source (self-hostable) | Free only for students |
| Free for personal use | Credit card required to start |
| Freemium with useful free limits | "Contact us" pricing |

---

## Adding a New Category

If you think a whole new category is needed:

1. Open an **Issue** first with title: `New Category: [Name]`
2. Explain why existing categories don't fit
3. We'll discuss and approve before you PR

---

## Pull Request Checklist

Before submitting, confirm:

- [ ] Resource has a working free tier
- [ ] Link is to the official site
- [ ] Description is one line, under 15 words
- [ ] No duplicate of existing entry
- [ ] Correct category selected
- [ ] PR title follows the format above

---

## Reporting Issues

Found a broken link or outdated resource? Open an issue with:

- **Title:** `[Broken Link] Tool Name` or `[Outdated] Tool Name`
- **Details:** What's wrong and the correct link if you know it

---

## Code of Conduct

Be respectful. This is a community resource maintained by volunteers. Constructive feedback only.

---

Thanks for making this better for developers everywhere 🌍
